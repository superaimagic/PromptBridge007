## 1. Coze Skill 平台规范速览

### 1.1 什么是 Coze Skill？

Skill 是扣子平台（coze.cn）的**"能力插件"**，允许创作者将个人方法论、专业知识和工具流程封装为可复用的 AI 技能，上传到技能商店供其他用户免费使用或付费购买。

### 1.2 Skill 标准文件结构

```
my_skill/                    # 技能文件夹（名称即 skill_id，全小写）
├── SKILL.md                 # 【必选】元数据 + 核心说明文档
├── assets/                  # 【可选】模板文件、静态资源（HTML/CSS/图片等）
│   └── template.html
├── references/              # 【可选】参考文档、API 规范、设计说明
│   └── api-spec.md
└── scripts/                 # 【可选】可执行代码（Python/Node.js）
    └── main.py
```

### 1.3 SKILL.md 标准模板

```markdown
---
name: 技能中文名称
skill_id: my_skill_id
version: 1.0.0
description: 一句话说明用途
tags: ["标签1", "标签2"]
---

# 技能名称

## 任务目标
- 解决什么问题
- 触发场景

## 执行流程
1. 步骤一
2. 步骤二

## 输入
- 必填参数：xxx
- 可选参数：xxx

## 输出
- 格式：Markdown / JSON / HTML
- 示例：[展示结果示例]

## 注意事项
- 约束条件
```

### 1.4 核心设计原则

- **提前固化可变环节**：预设风格文件、模板，避免 AI 每次从零推理
- **流程自动化**：将专业方法论写成分步骤指引
- **自适应匹配**：预设常用方案，超出预设时让 AI 自主发挥
- **稳定与速度**：通过预置样式和模板确保输出一致性

### 1.5 上架与变现流程

1. 在扣子编程中创建 Skill → 2. 编写 SKILL.md 和资源文件 → 3. 调试测试 → 4. 部署（设置 Logo/名称/简介）→ 5. 上架技能商店 → 6. 设置付费（免费 / 按月订阅）→ 7. 开通商户账户（支持支付宝/抖音支付）

## 3. Skill 适配评估标准

基于 Coze 平台的特点和 Skill 的设计理念，制定以下五维评估标准：

### 3.1 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **需求广度** | 30% | 目标用户群体大小、使用频率 |
| **独立完整性** | 25% | 是否可独立完成完整任务（不依赖其他 Agent） |
| **输出标准化** | 20% | 输出是否可模板化、可固化 |
| **付费意愿** | 15% | 目标用户是否有付费习惯和付费能力 |
| **本土化优势** | 10% | 是否有中国市场/平台特有知识 |

### 3.2 评分等级

- **S 级 (≥90 分)**：顶级商业化潜力，建议优先开发
- **A 级 (80-89 分)**：高商业化潜力，建议第二批开发
- **B 级 (70-79 分)**：中等潜力，观察市场需求后开发
- **C 级 (<70 分)**：建议暂不单独开发，可组合打包

## 5. Skill 封装改造方案

### 5.1 通用改造流程

当前提示词文件（"Agency Agent 人格"）与 Coze Skill 之间存在以下关键差异，需要针对性改造：

| 维度 | 当前状态 | 目标状态（Coze Skill） |
|------|---------|---------------------|
| **定位** | AI 角色人格定义（"你是谁"） | 任务执行工具（"你做什么"） |
| **结构** | 身份→使命→规则→交付物 | 目标→流程→输入→输出→约束 |
| **交互** | 对话式 Agent 人格 | 一次性任务执行 + 结果输出 |
| **模板** | 部分文件有嵌入模板 | 需提取到 assets/ 目录 |
| **独立性** | 强 | 需确保无需其他 Agent 协作 |

### 5.2 改造模板示例

以 `engineering-code-reviewer` 改造为例：

**原始结构（Agency Agent）**:
```
---
name: 代码审查员
---
## 🧠 你的身份与记忆
## 🎯 你的核心使命
## 🚨 你必须遵守的关键规则
## 📋 你的技术交付物
```

**改造后结构（Coze Skill）**:
```
my_skill/
├── SKILL.md          # 任务目标 + 执行流程
├── assets/
│   └── review-template.md   # 代码审查报告模板
├── references/
│   └── code-review-checklist.md  # 审查清单
```

```markdown
---
name: 代码审查与质量保障
skill_id: code_reviewer
version: 1.0.0
description: 自动审查代码质量，按阻塞/建议/改进三级分类输出专业报告
tags: ["开发工具", "代码质量", "工程"]
---

# 代码审查与质量保障

## 任务目标
- 对用户提交的代码进行专业审查
- 按三级分类输出：🔴阻塞 / 🟡建议 / 💭改进
- 提供可执行的具体修改建议

## 执行流程
1. 读取用户提交的代码
2. 按审查清单逐项检查
3. 分类问题严重级别
4. 生成结构化审查报告

## 输入
- `code`: 待审查的代码（必填）
- `language`: 编程语言（可选，自动检测）
- `focus`: 审查重点（可选：安全/性能/可读性/全部）

## 输出
- 格式：Markdown 审查报告
- 包含：问题分级列表、具体位置、修改建议、参考代码
```

### 5.3 高价值改造技巧

1. **预置模板到 assets/**：将原文件中的"技术交付物"提取为独立模板文件，AI 只需填充变量
2. **固化检查清单到 references/**：将"关键规则"转为结构化检查清单
3. **添加示例到输出区**：在 SKILL.md 的输出部分附上一个完整示例，提升用户信心
4. **参数化输入**：将"可选"行为改为显式参数，提升可控性
