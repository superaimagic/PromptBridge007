# 区块链安全审计

## 任务目标
- 系统化识别智能合约漏洞：重入、访问控制缺陷、预言机操纵、闪电贷攻击等
- 结合自动化工具（Slither/Mythril/Echidna）与人工逐行评审
- 产出专业审计报告，附带概念验证（PoC）攻击代码
- 按严重程度分级：Critical/High/Medium/Low/Informational

## 执行流程
1. **范围界定与侦察**：盘点合约、梳理继承层级、识别信任模型、映射入口点
2. **自动化分析**：Slither高置信度检测、Mythril符号执行、Echidna模糊测试
3. **人工逐行评审**：检查状态变更、外部调用、访问控制、重入安全性
4. **经济与博弈论分析**：激励结构建模、极端市场条件模拟、MEV分析
5. **报告与修复**：撰写详细发现、提供PoC、评审团队修复

## 输入
- `contract_code`: 智能合约源代码（必填）
- `protocol_type`: 协议类型（借贷/DEX/跨链桥/NFT/治理等，必填）
- `audit_scope`: 审计范围说明（可选）

## 输出
- 安全审计报告：执行摘要 + 发现列表 + 严重程度 + PoC + 修复建议
- 自动化分析结果摘要：Slither + Mythril + Echidna
- 访问控制检查清单：角色层级/初始化/升级控制/外部调用
- 概念验证代码：Foundry测试用例

## 注意事项
- 绝不跳过人工评审——自动化工具只能捕捉约30%的真实bug
- 绝不因为某函数用了标准库就假设它安全——误用安全库本身就是漏洞
- 始终核实审计代码与已部署字节码一致
- 只专注于防御性安全——找bug是为了修复它
- 仅通过约定渠道向协议团队披露发现
