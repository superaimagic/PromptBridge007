# UI/UX 设计系统生成器

## 任务目标
- 生成完整的设计系统：Design Token + 组件库 + 响应式框架 + 主题系统
- 创建可直接使用的CSS变量体系和布局框架
- 确保无障碍合规（WCAG AA标准）
- 提供清晰的开发者交付规格

## 执行流程
1. **分析项目需求**：审查品牌指南、用户界面需求、目标设备类型
2. **搭建设计基础**：色彩系统、排版层级、间距体系、阴影系统
3. **构建组件架构**：基础组件（按钮/表单/导航/卡片）+ 状态变体
4. **创建响应式框架**：断点策略 + 网格系统 + 容器系统
5. **开发者交付**：设计规格 + 组件文档 + 主题切换实现

## 输入
- `project_type`: 项目类型（Web应用/官网/后台/移动端，必填）
- `brand_info`: 品牌色彩和风格要求（可选）
- `target_devices`: 目标设备（手机/平板/桌面，可选）

## 输出
- Design Token系统：颜色/排版/间距/阴影/过渡的CSS变量
- 暗色主题Token：完整的暗色模式变量体系
- 基础组件样式：按钮/表单/卡片/导航的CSS代码
- 响应式布局框架：移动优先的断点和容器系统
- 主题切换组件：HTML模板 + JavaScript ThemeManager类
- 文件结构建议：css/ + js/ 组织方式

## 注意事项
- 所有设计均包含无障碍合规（最低WCAG AA标准）
- 采用移动优先方法设计响应式断点
- 使用语义化颜色命名，杜绝硬编码色值
- 建立8点间距系统保持垂直韵律
- 在创建单独页面之前先建立组件基础
