# 事件响应与应急管理

## 任务目标
- 制定技术事件应急响应方案
- 协助故障排查和根因分析
- 编写事后复盘报告（Postmortem）
- 建立预防措施和监控告警体系

## 执行流程
1. **事件识别**：根据告警和症状确定事件类型和影响范围
2. **应急响应**：止损→诊断→修复→恢复服务的标准流程
3. **根因分析**：5 Whys / 鱼骨图等分析工具
4. **复盘报告**：时间线 + 根因 + 影响 + 改进措施
5. **预防改进**：制定预防措施和监控增强方案

## 输入
- `incident_description`: 事件描述（必填）
- `affected_systems`: 受影响的系统/服务（必填）
- `severity`: 严重程度（P0/P1/P2/P3，必填）
- `timeline`: 事件时间线（可选）

## 输出
- 应急响应方案：止损步骤 + 诊断方法 + 恢复流程
- 根因分析报告：5 Whys分析 + 直接原因 + 根本原因
- 复盘报告（Postmortem）：完整时间线 + 影响评估 + 改进措施
- 预防建议：监控增强 + 流程改进 + 自动化方案

## 注意事项
- 响应优先级：止损 > 诊断 > 修复 > 恢复
- 事件处理过程全程记录，保留时间线
- 复盘要无责化——关注流程改进而非追责
- 改进措施要有明确的责任人和完成时间
