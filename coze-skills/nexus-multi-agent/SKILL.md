# NEXUS 多智能体编排系统

## 任务目标
- 为多AI智能体协作项目提供完整的编排框架
- 定义七阶段流水线：情报→策略→基础→构建→质量→上线→运营
- 建立智能体协调矩阵和交接协议
- 设置质量门禁，确保每阶段输出达标
- 管理跨职能团队的并行执行和依赖关系

## 执行流程
1. **第0阶段 - 情报与发现**：收集需求、分析市场、识别风险
2. **第1阶段 - 策略与架构**：制定技术架构、设计系统、确定路线图
3. **第2阶段 - 基础与脚手架**：搭建基础设施、配置环境、初始化项目
4. **第3阶段 - 构建与迭代**：分Sprint执行开发任务
5. **第4阶段 - 质量与加固**：测试、审查、性能优化
6. **第5阶段 - 上线与增长**：部署、监控、营销推广
7. **第6阶段 - 运营与演进**：持续运维、迭代优化

## 输入
- `project_type`: 项目类型（必填）
- `team_roles`: 团队角色配置（必填，如工程/设计/营销/产品/测试）
- `timeline`: 项目时间线（可选）
- `quality_requirements`: 质量要求（可选）

## 输出
- 项目流水线配置：七阶段详细任务分解
- 智能体协调矩阵：按部门的智能体名单和角色定义
- 交接协议：每次交接的上下文格式和验收标准
- 质量门禁清单：每阶段通过标准
- 风险管理计划：风险识别和应对措施

## 注意事项
- 核心原则：流水线完整性、上下文连续性、并行执行、证据高于口说
- 没过质量门禁的阶段不能推进
- 每次交接都带完整上下文，没有智能体冷启动
- 独立工作流同时跑，压缩工期
- 每个任务最多重试3次就升级
