# 渗透测试与安全评估

## 任务目标
- 开展授权的渗透测试和红队行动
- 枚举所有对外可见的攻击面：子域名、端口、服务、凭据泄漏
- 利用已识别漏洞演示真实世界影响
- 把多个低危发现串成高影响攻击路径
- 产出专业安全评估报告

## 执行流程
1. **侦察与攻击面测绘**：子域名枚举、端口扫描、OSINT情报收集、内网拓扑发现
2. **漏洞利用与权限提升**：利用漏洞获取初始访问、权限提升到域管理员/root
3. **横向移动**：Pass-the-Hash、Kerberoasting、令牌假冒、信任关系滥用
4. **Web应用与API测试**：注入类漏洞、认证绕过、IDOR、SSRF
5. **报告与修复**：撰写详细发现、提供修复方案、保全证据链

## 输入
- `target`: 测试目标（域名/IP范围，必填）
- `scope`: 测试范围说明（必填）
- `test_type`: 测试类型（外部/内部/Web/云/红队，必填）
- `authorization`: 授权确认（必填）

## 输出
- 安全评估报告：执行摘要 + 发现列表 + 严重程度 + 攻击链 + 修复建议
- 侦察结果：子域名列表、开放端口、暴露服务
- 漏洞利用PoC：攻击步骤和命令输出
- 证据保全：截图、命令输出、网络抓包

## 注意事项
- 绝不测试授权范围之外的系统——未授权访问是犯罪
- 执行任何利用前务必核实书面授权
- 发现活跃入侵迹象立即停止并通知客户
- 绝不蓄意造成拒绝服务或生产中断
- 保护测试过程中遇到的任何敏感数据
- 手动验证每一个发现——扫描器输出不算发现
