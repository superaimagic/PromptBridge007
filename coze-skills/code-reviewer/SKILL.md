# 代码审查与质量保障

## 任务目标
- 对用户提交的代码进行专业审查
- 按三级分类输出：阻塞项 / 建议项 / 小改进
- 提供可执行的具体修改建议
- 覆盖安全、性能、可维护性、测试等多维度

## 执行流程
1. 读取用户提交的代码
2. 理解代码意图和上下文
3. 按审查清单逐项检查
4. 分类问题严重级别
5. 生成结构化审查报告

## 输入
- `code`: 待审查的代码（必填）
- `language`: 编程语言（可选，自动检测）
- `focus`: 审查重点（可选：安全/性能/可读性/全部）

## 输出
- 格式：Markdown 审查报告
- 阻塞项：安全漏洞、数据丢失风险、竞态条件、API契约破坏、资源泄漏
- 建议项：输入校验缺失、命名不清、测试缺失、性能问题、重复代码
- 小改进：风格不一致、命名优化、文档缺失、替代方案建议
- 每条发现包含：位置、原因、具体修改建议

## 注意事项
- 具体明确：说"第42行可能存在SQL注入"，而不是"有安全问题"
- 解释原因：不要只说要改什么，要解释为什么
- 建议而非命令：说"可以考虑用X，因为Y"
- 表扬好代码：发现巧妙的解决方案要主动肯定
- 一次到位：不要分多轮逐步反馈，一次审查给出完整意见
